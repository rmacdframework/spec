# Contributing to RMACD Framework

Thank you for your interest in contributing to the RMACD Framework! This document provides guidelines for contributing to the project.

## How to Contribute

### Reporting Issues

If you find a bug, inconsistency, or have a suggestion:

1. **Search existing issues** to avoid duplicates
2. **Open a new issue** with a clear title and description
3. **Use labels** appropriately (bug, enhancement, documentation, question)

### Suggesting Enhancements

We welcome suggestions for improving the framework:

- **New operational patterns** — Real-world agent use cases
- **Implementation guidance** — Platform-specific integration guides
- **Compliance mappings** — Additional regulatory framework alignments
- **Permission profiles** — New profile templates for common agent types

### Submitting Changes

1. **Fork the repository**
2. **Create a feature branch** (`git checkout -b feature/your-feature-name`)
3. **Make your changes** with clear, descriptive commits
4. **Submit a pull request** with a detailed description

## Contribution Types

### Specification Changes

Changes to the core RMACD specification require:

- Clear rationale for the change
- Backward compatibility considerations
- Impact analysis on existing implementations

### Documentation Improvements

We especially welcome:

- Typo fixes and clarifications
- Additional examples
- Translations
- Implementation case studies

### Schema Updates

Changes to JSON schemas should:

- Maintain backward compatibility where possible
- Include updated examples
- Follow existing naming conventions

## Code of Conduct

Please read and follow our [Code of Conduct](CODE_OF_CONDUCT.md).

## Style Guidelines

### Documentation

- Use clear, concise language
- Include examples where helpful
- Follow existing formatting conventions
- Use tables for structured information

### JSON Schemas

- Use camelCase for property names
- Include descriptions for all properties
- Provide default values where appropriate
- Follow JSON Schema Draft 2020-12

## Recognition

Contributors will be acknowledged in:

- The CHANGELOG for their specific contributions
- The project README (for significant contributions)
